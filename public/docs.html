1 build() y buildAndInterpolate()

build(tagName, options, returnFirstChild, ctx)

Crea un elemento y ejecuta hidratación en este orden:

  1. 1 hydrateDirectives() - traducciones y data-each
  2. 2 hydrateIcons() - reemplaza [data-icon]
  3. 3 hydrateComponents() - [data-component]
  4. 4 hydrateEventListeners() - on-*, route-to 
import { build } from "@/core/dom";

const el = build("div", {
  className: "p-4 bg-slate-100 rounded",
  innerHTML: `<button class="btn">OK</button>`
}, true);

buildAndInterpolate(template, ctx, returnFirstChild, options)

Patrón típico: interpola primero, luego hidrata. Usado en todos los componentes. 

import { buildAndInterpolate } from "@/core/dom";
import { BaseComponent } from "@/core/types";

export class HelloComponent extends BaseComponent {
  render() {
    const template = `
      <div class="p-4 rounded border">
        <h2 class="font-bold">Hola {user.name|upper}</h2>
        <p class="text-sm">Edad: {user.age}</p>
      </div>
    `;
    return buildAndInterpolate(template, this);
  }
}

2 Interpolación y getValue()

Interpolación básica con {}

Ejemplos de paths:

{user.name}
{user.profile.email}
{items[0].title}
{language.{lang}} // interpolación interna

Comportamiento:

  • • Si no existe: deja {expr} sin sustituir
  • • Si es función: la ejecuta con apply()
  • • Soporta acceso a 'self' (global browser)
  • • Fallback a scope padre con '#'

Pipes / Filtros con |

Filtros globales disponibles:

upper lower → mayús/minús
if:clase → clase condicional
show hide → display: none
iif:true:false → ternario
undefined debug → utils

Ejemplos de uso:

<div class="p-2 {isActive|if:'bg-green-200'}">
  <h3>{title|upper}</h3>
  <div style="{isLoading|show}">Cargando...</div>
  <span>{status|iif:'✅ Activo':'❌ Inactivo'}</span>
</div>

Traducciones integradas

Sintaxis soportada:

{t:home.title}
{'home.title'} // literal
<span data-t="home.title"></span> // directiva

Comportamiento:

  • • Usa APP_CONFIG.i18n.t()
  • • Pasa el scope como contexto
  • • Soporta argumentos con :
  • • data-t se convierte a textContent

3 Control de Flujo con @if

!

Comportamiento especial con variables no definidas

Si la condición referencia una variable que no existe, el bloque @if se mantiene intacto para debugging.

Ejemplo básico

<div class="p-4 border rounded">
  @if(isAdmin)
    <span class="text-red-600 font-bold">Admin</span>
  @endif

  <span>Hola {user.name}</span>

  @if(user.hasNotifications)
    <div class="mt-2 p-2 bg-blue-100 rounded">
      Tienes {user.notificationCount} notificaciones
    </div>
  @endif
</div>

Con data-each

<div class="space-y-2" data-each="task in tasks">
  <div class="p-3 border rounded">
    <div class="font-medium">{task.title}</div>
    
    @if(task.completed)
      <span class="text-green-600 text-sm">✅ Completado</span>
    @endif
    
    <div class="text-xs text-slate-500">
      Índice: {index} | App: {#.appName}
    </div>
  </div>
</div>

Evaluación de condiciones

Las condiciones se evalúan con evaluateExpression() que usa with(ctx) { return expression; }

✅ Condiciones válidas

@if(isActive)
@if(user.role === 'admin')
@if(items.length > 0)

⚠️ Variable no definida

@if(noExiste)
→ Se mantiene el bloque

❌ Error de sintaxis

@if(user.name ==)
→ Se evalúa como false

4 DSL Compilado (template-compiler.ts)

DSL vs Template normal

El DSL compila a AST + función JavaScript, mientras que template.ts hace reemplazos de string. El DSL soporta @else y bucles @each más sofisticados.

Funciones del DSL

executeDSL(template, ctx) → compila y ejecuta
buildAndInterpolateDSL(template, ctx) → + hidratación

Sintaxis soportada

{expr} Interpolación
@if(cond) @else @endif Condicionales
@each(item in list) @endeach Bucles

Ejemplo completo con @each + @if/@else

<div class="p-4">
  <h2 class="font-bold mb-4">Usuarios ({users.length}+)</h2>

  @each(u in users)
    <div class="py-2 border-b flex items-center justify-between">
      <div>
        <div class="font-medium">{u.name|upper}</div>
        <div class="text-sm text-slate-500">{u.email}</div>
      </div>
      
      <div>
        @if(u.active)
          <span class="px-2 py-1 bg-green-100 text-green-800 rounded text-xs">Activo</span>
        @else
          <span class="px-2 py-1 bg-slate-100 text-slate-600 rounded text-xs">Inactivo</span>
        @endif
      </div>
    </div>
  @endeach
</div>

5 Hidratación (hydrate.ts)

hydrateIcons()

Reemplaza placeholders [data-icon] por SVG reales.

<i data-icon="zap" class="size-5 text-indigo-600"></i>
<i data-icon="heart" class="w-4 h-4 text-red-500"></i>
Se convierte en SVG

hydrateEventListeners() - Eventos reactivos

A) on-publish (Suscripción reactiva)

Sintaxis: on-publish="topic:scope:action:...extraArgs"

Acciones soportadas:
classname el.className = data
html el.innerHTML = data
json JSON.stringify
style:prop el.style[prop] = data
toggleclass:name el.classList.toggle
attr.name el.setAttribute
Ejemplos:
<pre on-publish="user-updated:local:json"></pre>
<div on-publish="theme:global:classname"></div>
<span on-publish="loading:local:toggleclass:animate-spin"></span>

B) route-to (Navegación)

<a route-to="/home">Home</a>
<a route-to="@routes.profile">Profile</a>
<button route-to="/admin/users">Admin</button>

• Añade listener click automático

• preventDefault() incluido

• @ resuelve desde el contexto

• Funciona en cualquier elemento

C) on-* (Eventos del DOM)

Modo publish:
<button on-click="publish:counter-inc:local:1">+1</button>
<input on-change="publish:filter:global:@searchTerm">

Publica {event, target, args} a PubSub

Modo handler:
<button on-click="onSave:@userId">Guardar</button>
<input on-input="onSearchChanged:@searchKey">

Llama a ctx[handler] con args resueltos

hydrateComponents() - Montaje de componentes

Proceso de hidratación:

  1. 1. Busca [data-component]
  2. 2. Resuelve el componente desde registry
  3. 3. component.init({parent: el})
  4. 4. component.render()
  5. 5. BaseComponent.bind()
  6. 6. Copia clases del placeholder
  7. 7. el.replaceWith(element)
  8. 8. component.mounted()

Ejemplo:

<div data-component="language-selector" class="w-full"></div>
<div data-component="user-profile" data-user-id="123"></div>

• data-* se convierte en props del componente

• Las clases se mantienen en el componente final

hydrateDirectives() - data-t y data-each

data-t (Traducciones)

<span data-t="home.title"></span>
<h1 data-t="t:nav.welcome"></h1>
  • • Usa APP_CONFIG.i18n.t()
  • • Se convierte a textContent
  • • Añade data-i18n-key

data-each (Repetición)

<ul data-each="u in users">
  <li>{u.name~upper} (#{index})</li>
  <li>App: {#.appName}</li>
</ul>
  • • Solo primer nivel (no anidados)
  • • ~ se convierte a | para pipes
  • • # accede al scope padre
  • • index automático

Orden de hidratación

Directivas → Iconos → Componentes → Eventos

Interpolación rota

{expr} se mantiene si no existe la variable

Pipes en data-each

Usar ~ en vez de | (se convierte automáticamente)
