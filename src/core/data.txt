Dirección: Del Objeto al DOM

async function hydrate(root: HTMLElement, ctx: ComponentContext) {
  const placeholders = root.querySelectorAll<HTMLElement>('[data-component]');
  
  for (const el of Array.from(placeholders)) {
    // Pasamos el MISMO ctx que recibió el padre
    const component = await loader.resolve(provider, ctx); 
    BaseComponent.mount(component, el, 'replace');
  }
}

private update() {
  if (!this.element) return;

  // 1. ¿Hay algo en el outlet que debamos salvar?
  const outlet = this.element.querySelector('#router-outlet');
  const savedContent = outlet ? Array.from(outlet.childNodes) : [];

  // 2. Render normal
  const newElement = this.render();
  
  // 3. Reinyectar lo salvado en el nuevo render
  const newOutlet = newElement.querySelector('#router-outlet');
  if (newOutlet && savedContent.length > 0) {
    savedContent.forEach(node => newOutlet.appendChild(node));
  }

  // 4. El resto de tu lógica de bind y replace...
  this.element.replaceWith(newElement);
  this.bind(newElement);
}

Si vienes de Angular (CDK) o React, aquí tienes la analogía para que termines de conectar los cables:

1. La analogía: Router-Outlet vs. Portals
CDK Portals: Son un sistema para proyectar dinámicamente una "pieza" de UI (un componente o un template) en un contenedor externo (PortalOutlet).

Router-Outlet: Es, en esencia, un PortalOutlet permanente. Es un hueco en tu Layout que dice: "Yo no sé qué va aquí dentro, pero el que tenga el mando (el Router) que me lo inyecte cuando quiera".

2. ¿Por qué esto es vital para tu arquitectura "Vanilla"?
Si usas un router-outlet, resuelves el problema de la "Masacre de Hijos" que mencionamos antes de dos formas:

A. Desacoplamiento de Ciclos de Vida
Cuando el Layout se actualiza (por su propio state), solo se refresca a sí mismo. Si el router-outlet está fuera de su lógica de render(), el componente de la ruta (la página actual) ni se entera, manteniendo su estado intacto.

B. Inyección Manual (Estilo Portal)
Tu Router actúa como el PortalHost. Cuando cambia la URL:

Busca el elemento #router-outlet en el DOM.

Limpia lo que había (llamando a destroy()).

Crea la nueva vista y hace un appendChild.

3. Cómo implementarlo de forma elegante
Para que tu sistema sea robusto, puedes crear una utilidad de "Teletransporte" (muy parecida a la de CDK) dentro de tu BaseComponent o tu Router:

JavaScript
// En tu Router
async renderView(route) {
  const container = document.getElementById('router-outlet');
  if (!container) return;

  // 1. Limpiar el "Outlet"
  this.currentView?.destroy();
  container.innerHTML = '';

  // 2. Resolver el componente (El "Portal")
  const view = await loader.resolve(route.componentProvider, this.ctx);
  
  // 3. Montar (Usando el método static que creamos)
  BaseComponent.mount(view, container, 'append');
  this.currentView = view;
}
4. ¿Y si el Layout tiene que ser reactivo?
Si el Layout necesita cambiar algo (ej: el color de la sidebar) y eso requiere re-renderizar, para no perder el "Portal" (la ruta), puedes usar la técnica de "Preservación de Nodo":

En tu BaseComponent.update(), antes de hacer el replaceWith, busca si el nodo que vas a reemplazar contiene al #router-outlet. Si es así, desengancha el contenido del outlet, haz el update, y vuelve a engancharlo. Es como mover un mueble antes de pintar la pared y volver a ponerlo en su sitio.
