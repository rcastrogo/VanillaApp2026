2. Pasar Children (Transfusión de Nodos)
Para que los hijos que escribes dentro del div original no se pierdan al renderizar el componente, debes realizar una "captura" antes de que el innerHTML del componente sobrescriba el contenido.

En el HTML:

HTML
<div data-component="Card">
  <button on-click="sayHi">Click me!</button> </div>
En tu lógica de Hidratación (hydrateComponents):
Para que esto funcione, tu función de hidratación debe mover los hijos del nodo original al nuevo contenedor del componente.

TypeScript
function hydrateComponents(el: HTMLElement, ctx: ComponentContext) {
  const componentName = el.getAttribute('data-component');
  if (!componentName) return;

  // 1. CAPTURA: Guardamos los hijos originales antes de limpiar el elemento
  const children = Array.from(el.childNodes);
  el.innerHTML = ''; 

  // 2. INSTANCIA: Creamos el componente (suponiendo que tienes un mapa de clases)
  const ComponentClass = ComponentRegistry[componentName];
  const instance = new ComponentClass(el); 

  // 3. RENDER: El componente genera su estructura interna
  const shadow = instance.render(); 
  el.appendChild(shadow);

  // 4. TRANSFUSIÓN: Buscamos dónde queremos meter los hijos originales
  const slot = el.querySelector('[data-slot]') || el; 
  children.forEach(child => slot.appendChild(child));

  instance.bind(el);
}
3. Ejemplo Práctico: El "Slot"
Si quieres tener control total de dónde caen los hijos, puedes usar un atributo marcador como data-slot.

Template del Componente Card:

TypeScript
render() {
  return build('div', { className: 'card-wrapper' }, `
    <div class="card-header">Título de la Card</div>
    <div class="card-body" data-slot>
       </div>
  `);
}
Resumen de la estrategia
Props: Usa this.element.getAttribute('data-prop-name'). Es simple, estándar y no requiere parseo complejo.

Children: En el proceso de hidratación, guarda los childNodes en una constante antes de renderizar el componente, y luego hazles un appendChild dentro del nuevo contenedor.

Esto permite que tus componentes sean reutilizables y que el HTML actúe como una "plantilla de configuración".

1. Captura y Pasaje en el Constructor (init)
Podemos modificar tu lógica de hidratación para que "secuestre" los hijos antes de que el componente tome el control del elemento.

TypeScript
function hydrateComponents(el: HTMLElement, ctx: ComponentContext) {
  const componentName = el.getAttribute('data-component');
  if (!componentName) return;

  // 1. CAPTURA: Guardamos los nodos que el usuario escribió en el HTML
  const children = Array.from(el.childNodes);
  
  // Limpiamos el elemento para que el componente empiece de cero
  el.innerHTML = '';

  // 2. INYECCIÓN: Pasamos los hijos al constructor
  const ComponentClass = ComponentRegistry[componentName];
  const instance = new ComponentClass(el, children); // <-- Pasamos el array de nodos

  // El componente ahora decide qué hacer con ellos en su render() o mounted()
}
2. Cómo recibirlos en el Componente
En tu clase base o en el componente específico, guardas esos hijos y decides dónde ponerlos. Esto es similar a cómo funciona el props.children en React o los slots en Web Components.

TypeScript
class MyCard extends BaseComponent {
  private children: Node[];

  constructor(el: HTMLElement, children: Node[]) {
    super(el);
    this.children = children;
  }

  render() {
    // Creamos la cáscara del componente
    const container = build('div', { className: 'card' }, `
      <div class="card-header">Título</div>
      <div class="card-content" id="slot"></div>
    `);

    // Inyectamos los hijos capturados en el lugar correcto
    const slot = container.querySelector('#slot');
    if (slot) {
      this.children.forEach(child => slot.appendChild(child));
    }

    return container;
  }
}
3. ¿Y las Props? (El enfoque "HTML-First")
Para las props, lo más sencillo es recuperarlas del el (el elemento original) durante el init. Así el componente nace ya sabiendo quién es.

TypeScript
class BaseComponent {
  protected props: Record<string, string> = {};

  constructor(public element: HTMLElement, public children: Node[] = []) {
    // Extraemos todos los data-attributes automáticamente
    const dataset = this.element.dataset;
    for (const key in dataset) {
      this.props[key] = dataset[key]!;
    }
  }
}
Por qué este enfoque es mejor:
Reactividad: Si el componente se re-renderiza (tu función update), ya tienes los children guardados en una variable de clase. No tienes que volver a buscarlos en el DOM.

Flexibilidad: El componente puede decidir no renderizar los hijos si una condición no se cumple (ej: un componente Acordeón cerrado).

Simplicidad: Mantienes el HTML declarativo pero la lógica de manipulación se queda dentro de la clase JS.

1. La Clase Base Evolucionada
TypeScript
export class BaseComponent {
  protected props: Record<string, string> = {};
  protected children: Node[] = [];

  constructor(public element: HTMLElement) {
    // 1. Capturar Props (de data-attributes)
    // dataset convierte data-user-id en userId automáticamente
    this.props = { ...this.element.dataset };

    // 2. Capturar Children antes de que se borren
    // Usamos Array.from para tener una copia real y no un live-list
    this.children = Array.from(this.element.childNodes);

    // 3. Limpiar el elemento original para el render
    this.element.innerHTML = '';
  }

  // Método que los hijos implementarán
  render(): HTMLElement | Node {
    return document.createElement('div');
  }

  // Tu lógica de bind, update, etc...
}

3. Ajuste en la Hidratación (hydrateComponents)
Tu orquestador solo necesita instanciar el componente y añadir el resultado del render al elemento original.

TypeScript
function hydrateComponents(el: HTMLElement, ctx: ComponentContext) {
  const name = el.getAttribute('data-component');
  const ComponentClass = ComponentRegistry[name];

  if (ComponentClass) {
    // El constructor de BaseComponent ya se encarga de capturar todo
    const instance = new ComponentClass(el);
    const view = instance.render();
    
    el.appendChild(view);
    instance.bind(el);
    instance.mounted?.();
  }
}
